# Contributing

Contributions are welcome. Please follow these guidelines.

## Reporting Issues

Use [GitHub Issues](https://github.com/pushberryfinn/laravel-domains/issues) to report bugs or request features. Search existing issues first to avoid duplicates.

## Submitting Pull Requests

1. Fork the repository and create your branch from `main`.
2. Install dependencies: `composer install`
3. Make your changes and add or update tests as needed.
4. Run the test suite and confirm it passes: `composer test`
5. Apply code style formatting: `vendor/bin/pint`
6. Submit a pull request with a clear description of the change and why it's needed.

## Running Tests

```bash
composer test
```

## Code Style

This project uses [Laravel Pint](https://github.com/laravel/pint) with the `laravel` preset.

```bash
vendor/bin/pint
```

## Versioning

This project follows [Semantic Versioning](https://semver.org/).
